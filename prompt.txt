FRUK Website v0 — Static GitHub Pages Tech Spec

1) Goal
Produce a single-page static website for Fertility Research UK (FRUK) that is:
	•	Easy to review in a meeting (v0 prototype)
	•	Easy to deploy on GitHub Pages
	•	Credible, modern, warm, and simple (not clinical, not salesy)
	•	No fundraising/donations functionality in v0

Deliver a working repo with index.html + small CSS/JS + assets.

⸻

2) Output deliverables
	1.	A Git repo containing:
	•	index.html
	•	styles.css
	•	script.js (minimal, optional but recommended for mobile nav + form UX)
	•	/assets/ folder (logo placeholder SVG, any icons)
	•	README.md with deployment steps for GitHub Pages

⸻

3) Site structure according to the text in file content.txt.

------

4) Visual design requirements
Tone: modern, warm, trustworthy, minimal. Avoid gendered clichés and overly clinical imagery.

Design constraints:
	•	Clean typography (system fonts ok; or one Google Font for headings + system body)
	•	Soft neutral background with subtle geometric motif (simple CSS shapes OK)
	•	Accessible contrast and readable sizing
	•	Mobile-first responsive layout

Brand tokens (implement as CSS variables):
	•	--bg, --text, --muted, --primary, --accent, --border
	•	Use these consistently so the palette can be changed quickly.

Components to implement (simple HTML/CSS):
	•	Sticky header with mobile hamburger menu
	•	Buttons (primary/secondary)
	•	Cards (support cards, people cards)
	•	Section headers

⸻

7) Technical constraints
	•	No frameworks (no React/Next/etc.)
	•	Plain HTML/CSS/JS only
	•	Prefer modern, semantic HTML (<header> <nav> <main> <section> <footer>)
	•	Keep JS minimal and non-invasive
	•	No build step required (repo should deploy as-is)

⸻

8) Accessibility and quality checklist
	•	Proper heading hierarchy: one <h1>, then <h2> per section
	•	Nav links work and highlight current section if easy (optional)
	•	Keyboard navigable menu
	•	Focus states visible
	•	Form labels correctly associated
	•	Lighthouse best-effort: strong accessibility/SEO

⸻

9) Repository layout

/
  index.html
  styles.css
  script.js
  README.md
  assets/
    logo.svg   (placeholder wordmark or simple mark)
    icons/     (optional)


⸻

10) README.md requirements (GitHub Pages deployment)
Include:
	•	How to preview locally (open index.html or python -m http.server)

⸻

11) Acceptance criteria (Definition of Done)
	•	One-page site with working anchored nav and mobile menu
	•	Looks good on mobile + desktop
	•	People cards render well with/without photos
	•	Form submits to placeholder Formspree endpoint and shows a success state
	•	Clean repo + clear README + ready for GitHub Pages

⸻

12) Non-goals for v0
	•	No donations / payment
	•	No CMS
	•	No blog/news
	•	No multi-language
	•	No analytics by default
